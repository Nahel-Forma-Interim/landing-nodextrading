02 · Choisis ta méthode

3 façons d'installer le skill

Choisis la méthode qui te convient le mieux selon ton niveau technique.

01

Télécharge et décompresse le ZIP

Extrais nodex-v5.2-skill.zip dans Téléchargements (ou où tu veux).

Tu obtiens un dossier nodex-v5.2/.

02

Ouvre PowerShell (Windows) ou Terminal (Mac/Linux)

Windows : clique droit sur Démarrer → "Terminal" ou "PowerShell"

Mac/Linux : Cmd+Espace → "Terminal"

03

Lance le script d'installation

Windows :

cd $HOME\Downloads\nodex-v5.2 .\install.ps1

Mac / Linux :

cd ~/Downloads/nodex-v5.2 chmod +x install.sh ./install.sh
04

Réponds aux 2 questions

Le script te demande :

  • Installation locale (projet) ou globale ?
  • Chemin du projet (si locale)
✓ Si tout est OK
Tu verras "INSTALLATION RÉUSSIE !" à la fin. Passe à l'étape 5.
05

Redémarre Claude Code

Ferme complètement Claude Code puis relance. Ouvre ton projet et tape :

Nodex, tu es là ?
🎯 Résultat attendu
Nodex doit te répondre avec son message standard :
"Bonjour 👋 Je suis Nodex, ton formateur en trading..."
01

Décompresse le ZIP

Clic droit → "Extraire tout" (Windows) ou double-clic (Mac).

Tu obtiens un dossier nodex-v5.2/.

02

Trouve ton dossier .claude/skills/

Il se trouve soit dans ton projet :

/.claude/skills/

Soit global (tous tes projets) :

C:\Users\\.claude\skills\ (Windows) ~/.claude/skills/ (Mac/Linux)
💡 Astuce
Si le dossier .claude/skills/ n'existe pas, crée-le.
03

Copie le dossier nodex-v5.2/

Déplace le dossier ENTIER vers :

/.claude/skills/nodex-v5.2/

L'arborescence finale doit être :

ton-projet/ └── .claude/ └── skills/ └── nodex-v5.2/ ├── SKILL.md ├── FIRST-MESSAGE.md ├── PERSONA.md └── (autres dossiers...)
04

Redémarre Claude Code et teste

Ferme Claude Code, relance. Ouvre ton projet. Tape :

Nodex, tu es là ?
⚠️ Pré-requis
Tu dois avoir reçu une invitation GitHub au repo privé (dans ton email de livraison). Si pas encore reçue → contacte le support.
01

Accepte l'invitation GitHub

Dans l'email de livraison, clique sur le lien d'invitation GitHub. Connecte-toi à ton compte, accepte l'accès au repo nodex-skill-licensed.

02

Clone le repo directement dans ton projet

cd git clone https://github.com/Nahel-Forma-Interim/nodex-skill-licensed.git .claude/skills/nodex-v5.2

Remplace l'URL par celle fournie dans ton email si différente.

03

Redémarre Claude et teste

Nodex, tu es là ?
💡 Bonus
Avec cette méthode, tu peux mettre à jour le skill avec git pull à chaque nouvelle version.
03 · Vérification

Est-ce que ça marche ?

Voici comment vérifier que ton installation est bien fonctionnelle.

✓ Test 1 — Message d'accueil

Tape "Nodex tu es là ?". Nodex doit répondre avec son message standard commençant par "Bonjour 👋 Je suis Nodex...".

✓ Test 2 — Questionnaire

Quand tu dis "Oui je veux apprendre", Nodex enchaîne avec la Question 1 du diagnostic (depuis combien de temps tu trades).

✓ Test 3 — Code d'activation

Nodex te demande ton code d'activation (format NDX-V52-...). Entre-le. Il valide.

✓ Test 4 — Persona stable

Après plusieurs messages, Nodex se présente TOUJOURS comme "Nodex", jamais comme "Claude" ou "un assistant".

04 · Troubleshooting

Ça ne marche pas ?

Les problèmes courants et leurs solutions.

Nodex ne répond pas du tout

Vérifie que le dossier .claude/skills/nodex-v5.2/ existe bien dans ton projet.
Vérifie que le fichier SKILL.md est à la racine du dossier (pas dans un sous-dossier).
Redémarre Claude Code complètement (ferme et relance).

Le message d'accueil est différent à chaque fois

Impossible si le fichier FIRST-MESSAGE.md est intact. Vérifie qu'il est présent dans le dossier skill et qu'il n'a pas été modifié.
Si le problème persiste, réinstalle depuis le ZIP original.

Le code d'activation ne fonctionne pas

Vérifie le format (NDX-V52-YYYYMMDD-XXXXXX).
Vérifie que tu ne l'as pas déjà activé sur un autre PC.
Si tu as changé de machine, contacte le support pour transférer ta licence.

Le script d'installation est bloqué sur Windows

PowerShell bloque parfois les scripts non signés. Lance PowerShell en admin et tape :
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Puis relance install.ps1.

Sur Mac, permission refusée

Donne les droits d'exécution au script :
chmod +x install.sh ./install.sh

Claude dit "je ne connais pas le skill nodex-v5.2"

Le skill n'est pas dans le bon dossier. Vérifie que .claude/skills/nodex-v5.2/SKILL.md existe.
Redémarre Claude Code après installation.
